Docker 命令

Docker容器管理命令集合，包括镜像构建、容器运行、网络管理、数据卷等完整操作指南

镜像管理

构建和拉取

构建镜像

docker build -t [image-name]:[tag] .

从 Docker Hub 拉取镜像

docker pull [image-name]:[tag]

推送镜像到仓库

docker push [image-name]:[tag]

给镜像打标签

docker tag [source-image] [target-image]

查看和删除

列出本地镜像

docker images

查看镜像详情

docker inspect [image-name]

删除镜像

docker rmi [image-name]

删除未使用的镜像

docker image prune

容器管理

运行和控制

运行容器

docker run -d --name [container-name] [image]

交互式运行

docker run -it [image] /bin/bash

端口映射运行

docker run -p [host-port]:[container-port] [image]

挂载数据卷

docker run -v [host-path]:[container-path] [image]

容器操作

启动容器

docker start [container-name]

停止容器

docker stop [container-name]

重启容器

docker restart [container-name]

进入容器

docker exec -it [container-name] /bin/bash

查看信息

列出运行中的容器

docker ps

列出所有容器

docker ps -a

查看容器日志

docker logs [container-name]

实时查看日志

docker logs -f [container-name]

清理操作

删除容器

docker rm [container-name]

强制删除运行中的容器

docker rm -f [container-name]

删除所有停止的容器

docker container prune

查看容器资源使用

docker stats [container-name]

Docker Compose

基本操作

启动服务

docker-compose up -d

停止服务

docker-compose down

重启服务

docker-compose restart

查看服务状态

docker-compose ps

高级操作

构建并启动

docker-compose up --build

查看日志

docker-compose logs -f

执行命令

docker-compose exec [service] [command]

扩展服务

docker-compose up --scale [service]=[number]

网络管理

网络操作

列出网络

docker network ls

创建网络

docker network create [network-name]

连接容器到网络

docker network connect [network] [container]

删除网络

docker network rm [network-name]

数据卷管理

列出数据卷

docker volume ls

创建数据卷

docker volume create [volume-name]

查看数据卷详情

docker volume inspect [volume-name]

删除未使用的数据卷

docker volume prune

常用示例

Web 服务

运行 Nginx

docker run -d -p 80:80 nginx

运行 Apache

docker run -d -p 8080:80 httpd

运行 Node.js 应用

docker run -d -p 3000:3000 node:16-alpine

数据库

运行 MySQL

docker run -d -p 3306:3306 -e MYSQL_ROOT_PASSWORD=password mysql

运行 PostgreSQL

docker run -d -p 5432:5432 -e POSTGRES_PASSWORD=password postgres

运行 Redis

docker run -d -p 6379:6379 redis
